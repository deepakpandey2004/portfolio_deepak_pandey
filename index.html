Attendify

Face-recognition based event attendance system

A production-grade attendance system built to eliminate proxy attendance — each account is tied to a unique face using dlib's 128-dimensional facial encoding, so no two accounts can share the same face. Attendees check in through a peer-to-peer selfie verification flow with live camera capture and GPS-based reverse geocoding, producing a timestamped, watermarked selfie with the exact address. The entire attendance lifecycle — logout reminders, auto-logout, half-day marking, and weekend-aware leave tracking — runs on automated APScheduler jobs.

FastAPIPostgreSQL dlibAPScheduler DockerJWT
The hard part: enforcing one-account-per-face reliably at scale, and keeping location data accurate with a LocationIQ → BigDataCloud fallback when the primary geocoding API failed.

SpeakU v2.0

Real-time language exchange platform

A Lingbe-inspired platform that connects language learners for live conversation practice — WebSocket-based matchmaking pairs users, and WebRTC handles the peer-to-peer voice calls. Version 2.0 adds TURN server support to keep calls connecting reliably across restrictive networks, a session countdown timer to manage call duration, and Redis for faster session state handling. FastAPI powers the backend, with Supabase/PostgreSQL storing users and call history.

FastAPIWebSocket WebRTCRedis Supabase
The hard part: resolving WebRTC "glare" conditions — where both peers try to initiate a call offer simultaneously — by adding proper offer/answer collision handling, plus TURN servers to keep calls alive behind strict NATs/firewalls.
